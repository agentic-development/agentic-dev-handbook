## Overview

[[agentic-engineering|Agentic development]] is not a single framework but a combination of frameworks that are native to AI-assisted engineering. The following principles define what we mean when we talk about agentic development — and what separates it from simply using AI to write code.

## The Principles

### 1. Code Will Be Mostly Developed by Agents

The role of the human developer shifts from writing code to directing, reviewing, and orchestrating the agents that produce it. The developer becomes the architect; the agent becomes the builder.

### 2. Context Is the Biggest Constraint

In the agentic era, the bottleneck is no longer typing speed — it is [[context-engineering|context]]. Building a well-structured backlog that aligns with business needs and can be consumed by agents is the biggest challenge teams will face.

### 3. Well-Designed Specs Are Non-Negotiable

Agentic development expects well-designed specifications as proposed by frameworks like BMAD or Spec-Driven Development. Agents need structured, unambiguous inputs to produce reliable outputs.

### 4. The Entire SDLC Is Rethought

Agentic development rethinks the complete Software Development Lifecycle — from project scoping and work planning to execution and deployment. Every phase is redesigned for human-agent collaboration.

### 5. Teams Are Made of Agents and Humans

Software will be developed by teams that explicitly include both agents and humans. This is not a metaphor — it is a design principle that affects tooling, workflows, and team structure.

### 6. Enterprise Accountability by Design

Enterprise software development requires accountability and governance by design, even when developed by agents. Enterprise code needs a human sign-off, explicitly or implicitly, at every critical juncture.

### 7. Vibe Coding Is Not Agentic Development

[[vibe-coding]] is primarily an individual method of coding — rapid, exploratory, and unstructured. Agentic development is a team discipline with governance, repeatability, and accountability built in.

### 8. New Expertise Paths Are Required

The agentic era will deeply affect software engineering skill development. New ways of building deep software engineering expertise will be required — skills like [[context-engineering]], specification writing, and agent orchestration.

### 9. Code Is Ephemeral

In the agentic era, code is largely ephemeral. Agents should be able to "re-compile" applications from specs alone with minimal human intervention. The spec, not the code, becomes the durable artifact.

### 10. Best Practices Expand, Not Disappear

The agentic development movement does not replace but expands the software engineering best practices that are a cornerstone of modern software development. Testing, code review, architecture — these matter more, not less.

## A Note on Market Confusion

There is widespread confusion in the market between "shipping code" and "developing software." Shipping code is a solved problem — LLMs can generate syntactically correct code at scale. Developing software — understanding requirements, managing complexity, ensuring reliability, governing change — remains a deeply human challenge that agentic development addresses through structured collaboration between humans and agents.
